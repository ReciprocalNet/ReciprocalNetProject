Instructions to integrate JAXM with JWSDP 1.2 and later 

  • Copy jaxm-runtime.jar, jaxm-api.jar and dom4j.jar from {JAXM_HOME}/lib to {JWSDP_HOME}/common/lib.

  • Create a folder called jaxm-provider under {JWSDP_HOME}/webapps and unjar {JAXM_HOME}/jaxm/jaxm-provider.war into it.

    cd {JWSDP_HOME}/webapps/jaxm-provider
    jar xvf {JAXM_HOME}/jaxm/jaxm-provider.war

  • Copy all wars under samples, i.e. {JAXM_HOME}/samples/*.war and {JAXM_HOME}/tools/jaxm-provideradmin.war to {JWSDP_HOME}/webapps

  • Edit {JWSDP_HOME}/conf/tomcat-users.xml and add a user with the role "provider". This is the login that should be used to login to the provider admin tool.

  • Start tomcat from {JWSDP_HOME}/bin folder

  • Point your browser to http://localhost:8080/jaxm-docs/. You will find the documentation here. You can also run samples from this page.
Note: {JAXM_HOME} and {JWSDP_HOME} are the installation directories of JAXM and JWSDP respectively.
